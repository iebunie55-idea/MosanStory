# 모산초등학교 동화 만들기 수업용 PWA 설계

## 요약

기존 부여 AI STORY 키오스크 앱을 모산초등학교 3~6학년 학생들이 태블릿으로 사용하는 수업용 PWA로 확장한다. 현재의 6쪽 동화 구성, 따뜻한 3D 동화책 그림 스타일, 어린이 친화적인 한국어 문체는 기본값으로 유지한다.

1차 버전의 초점은 수업 진행 안정성이다. 학생은 제공된 아이디로 로그인하고, 같은 아이디는 다른 태블릿에서 재사용할 수 없다. 완성된 동화는 배경음악과 함께 앱에서 감상하고, 그림이 포함된 6쪽 동화책 PDF로 다운로드한다. Google Drive 업로드는 앱 밖에서 학생 또는 교사가 수동으로 진행한다.

## 수업 흐름

- 학생은 태블릿에서 PWA URL에 접속하거나 홈 화면에 추가된 앱을 연다.
- 첫 화면은 `모산초등학교 3~6학년 동화 만들기` 맥락이 분명히 보이도록 바꾼다.
- 학생은 교사가 제공한 수업용 아이디를 입력한다.
- 서버는 아이디가 미사용 상태이거나 같은 태블릿의 진행 중 세션일 때만 로그인을 허용한다.
- 로그인 후 학생은 기존 흐름을 따라 주인공, 성격, 배경, 기승전결 사건을 고른다.
- 앱은 기존 AI 생성 및 로컬 fallback 방식을 활용해 6쪽 동화를 만든다.
- 앱은 기존 그림 생성 방향을 활용해 각 쪽에 사용할 동화책 이미지를 만든다.
- 결과 화면에서는 배경음악 재생/정지, 페이지 넘김, PDF 다운로드를 제공한다.
- PDF 다운로드가 성공하면 해당 아이디는 완료 상태로 처리할 수 있다.

## 아키텍처

배포 형태는 태블릿 접속에 적합한 웹앱/PWA로 한다. 운영 환경은 Next.js 호스팅과 서버 DB 조합을 사용한다. 현재 기준 추천 조합은 Vercel + Supabase이지만, 구현 직전 Supabase 최신 문서와 보안 권장사항을 확인한다.

- 현재 Next 앱을 학생용 UI의 기반으로 유지한다.
- 로그인, 아이디 사용 상태, 완료 상태는 서버 DB에서 관리한다.
- 브라우저에는 로그인 성공 후 발급받은 세션 토큰만 저장해 새로고침과 탭 복구를 허용한다.
- AI API 키는 브라우저에 노출하지 않고 서버 API 또는 배포 가능한 프록시 계층에서만 사용한다.
- 기존 `output: "export"` 정적 export 전제는 서버 API와 PDF 생성이 가능하도록 재검토한다.
- PWA manifest, 아이콘, 모바일 viewport, 홈 화면 이름을 추가한다.
- 그림과 PDF 생성은 학생 태블릿 성능과 네트워크 실패 가능성을 고려해 진행 상태와 재시도 흐름을 제공한다.

## 로그인과 교사용 초기화

학생 아이디는 교사가 미리 준비한다. 예시는 `mosan-001`, `mosan-002` 형식이다. 1차 버전에서는 학생 이름, 비밀번호, 개인 계정 정보를 받지 않는다.

아이디 상태:

- `unused`: 아직 사용하지 않은 아이디
- `active`: 한 태블릿에서 진행 중인 아이디
- `completed`: PDF 다운로드까지 끝난 아이디

로그인 규칙:

- `unused` 아이디는 로그인을 허용하고 `active`로 바꾼다.
- `active` 아이디라도 같은 세션 토큰이면 이어 하기를 허용한다.
- `active` 아이디가 다른 태블릿에서 입력되면 이미 사용 중이라는 안내를 보여준다.
- `completed` 아이디는 완료된 아이디라는 안내를 보여주고 재사용을 막는다.

교사용 초기화:

- `/teacher` 같은 간단한 교사용 화면을 추가한다.
- 교사용 비밀번호 또는 관리자 토큰으로 보호한다.
- 아이디 목록, 상태, 완료 시각을 확인할 수 있게 한다.
- 실수 로그인, 태블릿 꺼짐, 수업 중 오류가 있을 때 특정 아이디를 `unused`로 되돌릴 수 있게 한다.
- 진행률 통계, 학생별 상세 분석, Drive 일괄 업로드는 1차 범위에서 제외한다.

## 동화, 그림, 음악, PDF

동화 구성:

- 기존 6쪽 구조를 유지한다: 소개, 발단, 전개, 절정, 해결 행동, 결말과 배운 점.
- 기존의 쉬운 낱말, 존댓말, 어린이 안전 표현 규칙을 유지한다.
- AI 생성 실패 시 로컬 동화 엔진으로 완성하는 fallback을 유지한다.
- 모산초등학교 3~6학년 수업에 맞게 화면 문구와 안내 문장을 다듬는다.

그림 스타일:

- 기존의 따뜻한 3D 애니메이션 동화책 느낌을 유지하거나 가볍게 보완한다.
- 주인공의 외형, 색상, 의상, 배경 분위기가 6쪽 전체에서 일관되도록 한다.
- 그림 안에는 글자, 로고, 워터마크, 특정 저작권 캐릭터 느낌을 넣지 않는다.

배경음악:

- 음악은 앱의 결과/감상 화면에서만 제공한다.
- PDF에는 음악을 넣지 않는다.
- 태블릿 브라우저의 자동재생 제한을 고려해 기본은 꺼진 상태로 두고, 학생이 버튼을 눌렀을 때 재생한다.
- 1차 버전은 저작권 문제가 없는 번들 배경음악 1곡을 사용한다.

PDF:

- 완성본은 표지 + 6쪽으로 구성된 PDF 한 파일이다.
- 각 쪽은 생성된 그림과 해당 쪽의 동화 문장을 함께 보여준다.
- 표지 또는 하단에 모산초등학교, 수업용 아이디, 생성일, 제목을 넣는다.
- 파일명은 `mosan-001-story-2026-06-06.pdf`처럼 안정적인 형식으로 만든다.
- Google Drive 업로드는 다운로드 후 수동으로 진행한다.

## 테스트와 승인 기준

- 미사용 아이디로 로그인하면 진행 중 상태가 된다.
- 같은 태블릿에서 새로고침해도 같은 아이디로 이어 할 수 있다.
- 다른 태블릿에서는 진행 중 아이디를 재사용할 수 없다.
- 완료된 아이디는 다시 시작할 수 없다.
- 교사용 화면에서 특정 아이디를 초기화하면 다시 사용할 수 있다.
- AI 생성 실패 시에도 로컬 엔진으로 6쪽 동화가 완성된다.
- 결과 화면에서 배경음악을 재생하고 정지할 수 있다.
- PDF에는 표지와 6쪽 그림/본문이 모두 포함된다.
- PDF의 한국어 본문은 태블릿 화면과 인쇄 환경에서 읽을 수 있다.
- PWA가 태블릿 홈 화면에 추가 가능한 메타데이터를 가진다.

## 1차 범위 밖

- Google Drive 직접 업로드 또는 Google OAuth 연동
- 네이티브 iOS/Android 앱스토어 배포
- 학생 개인 계정, 비밀번호, 개인정보 프로필
- 교사용 상세 통계 대시보드
- PDF 안에 배경음악 삽입
- 교사용 PDF 일괄 다운로드 또는 Drive 일괄 업로드
